The following examples show some "cut and paste" blocks of code that can be added to the HTML source of your web pages to include the DateClick popup calendar. Before including the example code, be sure to add a reference to the DateClick CSS and Javascript files in the <HEAD> section like so:

<script language="javascript" src="choosedate.js"></script>
<link type="text/css" rel="stylesheet" href="calendar.css">


Beginning Date
  		Standard Calendar-No Options

This basic calendar example shows a bare minimum script block to get DateClick on the page. The input field to the left is called datBegin. DateClick must be notified as part of the configuration string that this is the field you want it to populate when a date is clicked. The code needed for this is shown here. If you already have an existing <FORM> in your page, remove the FORM lines below before loading the page, and change the name of the field from document.frm.datBegin to reflect the name of your form (e.g. document.YourForm.datBegin). Remember, FORM tags can not be nested. 

<form name="frm">
<input type="text" name="datBegin" size="10">
&nbsp;
<script language=javascript>
	var basicCal = new calendar("FIELD:document.frm.datBegin");
	basicCal.writeCalendar();
</script>
</form>
Ending date:
  		Standard Calendar-Alternate Date Format

The Basic version of the calendar can be set to accept two alternate date formats other than the default mm/dd/yyyy most commonly used in the US. By setting the FORMAT key in the configuration string as shown below, you can specify a 1 for dd/mm/yyyy or a 2 for yyyy/mm/dd (default is 0). Also notice that this calendar object must be named differently to distinguish it from the example above. The DELIMITER option is also passed to give the format a dd-mm-yyyy look. 

<form name="frm">
<input type="text" name="datEnd" size="10">
&nbsp;
<script language=javascript>
	var basicCal2 = new calendar("FIELD:document.frm.datEnd;FORMAT:1;DELIMITER:-;");
	basicCal2.writeCalendar();
</script>
</form>
New In Standard--Different Languages

The months and days of DateClick are completely configurable. Rather than the default Jan, you could configure each month's name to be the complete word, e.g. January. Or "Janvier" for that matter. This is done by modifying the Month and Day name arrays before writing the calendar. This calendar string also sets the date format to dd/mm/yyyy, the delimiter to '-' (thus making the date format dd-mm-yyyy), and hides leading zeros. The latter two options are Pro/Developer version only. 

<form name="frm">
<input type="text" name="datDOB" size="10">
&nbsp;
<script language=javascript>
	var proDOB = new calendar("FIELD:document.frm.datDOB;ZEROS:false;" +
		"DELIMITER:-;FORMAT:1;DIR:primoris;");
	proDOB.aryMonths = Array("Janvier", "Février", "Mars", "Avril", 
		"Peut", "Juin", "Juillet", "Août", "Septembre", "Octobre", 
		"Novembre", "Décembre");
	proDOB.aryDays = Array("Sonntag", "Montag", "Dienstag", "Mittwoch", 
		"Donnerstag", "Freitag", "Samstag");
	proDOB.writeCalendar();
</script>
</form>
Additional Options for Pro and Developer Versions
The above options are the only ones that can be set in the STANDARD version of DateClick. Below are some examples of some of the many other options that are configurable in the PROFESSIONAL and DEVELOPMENT versions, available for purchase from Primoris Software.


Pro/Developer Versions-Extreme Options

In addition to being licensed for commercial use, the Professional and Developer versions of DateClick are highly configurable. The calendar can be moved when dragged with a mouse, it can stay open after a date is clicked, the weeks can begin on Monday, the calendar can cast a drop shadow in compatible browsers, the year and month flyouts can be configured to appear on a mouseover or not at all, and more! 

<form name="frm">
<input type="text" name="datDeparture" size="10">
&nbsp;
<script language=javascript>
	var Departure = new calendar("FIELD:document.frm.datDeparture;DIR:primoris;" +
		"MOUSEOVER:false;BEGINONMONDAY:true;SHADOW:true;CLOSE:false;MOVEMODE:1;");
	Departure.writeCalendar();
</script>
</form>
