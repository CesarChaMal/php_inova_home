Overview

Date fields on web input forms have long been a stumbling block for users and developers alike.  Users, who as we know come in all different shapes and sizes, are used to their own particular date formatting.  When entering data on the web, most find themselves having to choose between a dash or a slash, european or american formatting, etc., many times with frustrating results.  If you are developer giving a demonstration of your web application, or a merchant trying to make a sale, frustrating the user can cost you well-deserved sales. 

On the flip-side, developers then must solve their own problems with date input fields.  Usually you have to put these results into a database.  This means it has to be in a specific format.  What if it's an invalid date?  Even with your own validation routines, there is too much potential to raise an error.  Most software developers have long ago learned to add a pop-up calendar into their form to facilitate date picking, but until now there has been no comparable input control for web pages. 

DateClick is a "drop-in" web component written entirely in client-side Javascript.  With no unnecessary server-side code, the DateClick pop-up calendar can be called from any type of web page--HTML, ASP, ASP.NET, PHP, JSP, etc..  Any page that will be viewed in a browser will load DateClick. 

The DateClick icon will normally reside next to one or more date entry fields on your web entry form.  The calendar can be configured to appear either when the icon detects a click event or a mouse-over (Pro version).  Without any round-trips back to the server, the calendar appears using cross-browser compatible DHTML.  The appearance of the calendar is highly configurable with CSS and design-time options that are passed to the calendar upon creation.  Configuration options such as non-English display, begin on Monday, international date formatting, custom delimiters, and many more are available. 

DateClick has been designed to allow the selection of any date with as few clicks as possible, and was designed from the ground up with the most versatile navigation of any comparable product.  A date as far away as 30 or 40 years can be chosen with a fraction of the amount of clicking on traditional GUI calendars thanks to the year fly-out.  Any month of the year is one click away on the month fly-out in addition to the more traditional "next month", "previous month" navigaton.

Once a date has been selected, it is passed back to the input field of your choice in the format you specify.  Thus a normally frustrating and time consuming operation can be completed in as little as a single click! 

 

What's the difference between Standard (free) and Pro/Developer?

DateClick is geared towards developers of complicated user interfaces and web pages, but we believe the casual user should be able to have powerful user interface design as well. DateClick Standard is free for non-commercial use. This includes blogs, non-profits, sites run as a hobby, open source, etc.. For the professional user, DateClick Pro and Developer include many powerful enhancements (see the list of options below) and can be licensed for use on pages that generate sales and ad revenue. The Developer version can even be included in your own distributable web applications royalty free! See the Primoris Software website for more details on the commercial versions.

Adding DateClick to Your Web Page

DateClick is an object-oriented JavaScript component designed to be added to your page with minimal effort.  Adding one or more pop-up calendars to your page is as easy as creating the DateClick object and calling the method to write that object into the browser's window.  Even if you are not familiar with Javascript syntax, you can copy and paste the examples into your own design with no special programming experience!  A summary of adding the code to your page follows:

1. Add a reference to the script file that contains the object definition in the <head> tags of your web page.  "<script language='javascript' src= 'choosedate.js'></script>". Also add a CSS link to the style page to define the look of the calendar. <link type="text/css" rel="stylesheet" href="calendar.css">
2. Create a variable and set it to a new instance of the object. Notice the string in quotation marks that is passed to the method.  This is called the configuration string, and is how many internal settings of the calendar are set.  In this case, the FIELD parameter is specified.  Although this is not required to add the calendar to the page, it is required if you want to pass the chosen date value into a form field.  The field name should be the full document path to the field, including the form name.
var cal1 = new calendar("FIELD:document.frm.datBegin;");  This assumes that you have a form on your page named "frm" and an input field named datBegin.  If this is not the case, add the code to include this on your page, or change the name of the form to match your existing set up.
3. Call the writeCalendar() method of the calendar instance you created in the location you want the icon to appear (this is using the default "document flow" positioning--"absolute" positioning is also available). 
cal1.writeCalendar();
4. The supporting .CSS file (usually calendar.css) and .JS file (choosedate.js) should be in the same directory as the page that DateClick has been added to.  The graphic files should by default be in a subdirectory called "basic". 

Pro: In the professional version, you can specify the image directory and class name prefix.  With this feature, you can radically change the look of the DateClick popup calendar.  By default, this will be "basic", but by sending a "DIR" parameter in the configuration string you can set this on a per-object basis.  This is especially useful if you need to have more than one style of calendar on one page. 

 

Different Languages and Date Formats

DateClick can be set to return dates in 3 different formats.  The default format is m/d/yyyy.  By setting the FORMAT property in the configuration string to 1 this will be in d/m/yyyy format, and by setting it to 2 DateClick will return values in yyyy/m/d format.

Although DateClick was programmed in America in English, the calendar can be displayed in almost any format in almost any language.  You can change the format to the 3 formats listed above, and you can also specify whether leading 0's should be added and what delimiter should be used between the month, day, and year.  This will yield almost any conceivable date format, such as yyyy-mm-dd, mm.dd.yyyy, dd-mm-yyyy, etc.. 

 

Configuring Day and Month Displays

The way DateClick displays the names of the months and days is also configurable.  Instead of display the default of S, M, T, etc. for the days, you can configure it to show Sun, Mon, Tue, ..., or Sonntag, Montag, Dienstag,... This is accomplished by overriding the default ary_days property.  Do this in Javascript directly after creating the calendar object.  An example would be: 

<script language=javascript> 
	var myCal = new calendar("FIELD:document.formName.BeginDate;"); 
	myCal.aryDays = Array("Sonntag", "Montag", "Dienstag", "Mittwoch", "Donnerstag", "Freitag", "Samstag"); 
 myCal.aryMonths = Array("Janvier", "Février",
		"Mars", "Avril", "Peut", "Juin", "Juillet", "Août", "Septembre", "Octobre", "Novembre",
		"Décembre");
myCal.writeCalendar();
</script>

The above example will create a calendar that will display German dates in the day header row and French months. Notice that when the myCal variable is declared, a string is passed that gives the FIELD name.  This is the only required configuration parameter.  It tells DateClick what input field to pass the date back to.

 

Starting Day of the Week

Pro: The DateClick calendar defaults to a week beginning with Sunday, however the Professional version and above can be configured to begin the week display on monday by passing a BEGINONMONDAY value of 1 in the configuration string.

 

The Configuration String

When the DateClick calendar is created, a string is passed in the constructor to configure various options.  The FIELD option is required--this tells DateClick which form field it should pass the selected date value to.  See the configuration string passed in the following example:

<form name="formName">
	<input type="text" name="DateOfBirth">





	<script language= javascript> 
		var myCal = new calendar("FIELD:document.formName.DateOfBirth;");
		myCal.writeCalendar();
	</script>
</form>

In this example, a DateClick object is created called myCal.  It is created by instantiating a calendar object (This is the name of the DateClick calendar).  In the parenthesis after calendar, a string is passed--that is a "string" of characters enclosed in quotation marks.  The options in a configuration string are passed in exactly the same way as CSS options.  There is a variable name followed by a colon, then the value of the field and a semi-colon.  In this case, the variable is "FIELD", and the value is "document.formName.DateOfBirth".  document is the Javascript reference to the web page you are adding DateClick to--this will always be just document.  formName is the name of the form that encloses the DateOfBirth field, which is a text type of an HTML input.

Pro: The Professional and Developer editions of DateClick have some very flexible options that can control how and when the DHTML pop-up calendar is displayed.  A full reference can be found in the table below.

Options available in the Standard, Pro, and Developer versions of DateClick
FIELD
REQUIRED		 (no default) This value must always be explicitly set.  FIELD must equal the full DOM path to the form field DateClick will populate with the selected value.  The path will always be preceded by 'document', then the name of the form, then the name of the field.

e.g. FIELD:document.forms[0].txtEndDate;
FORMAT 0 Sets the format of the returned date.  Note that in the Pro and Developer versions, this setting is overridden by what is set in MASK!
0 = mm/dd/yyyy
1 = dd/mm/yyyy
2 = yyyy/mm/dd

e.g. FORMAT:1;
DELIMITER / Sets the delimiter between month day and year.  This allows for date formats such as dd.mm.yyyy and yyyy-mm-dd. Note that in the Pro and Developer versions, this setting is overridden by what is set in MASK!


e.g. DELIMITER:-;
HIDELISTBOXES false Obsolete--this is included for backward compatibility.   

e.g. HIDELISTBOXES:true;
Options available only in Pro and Developer
MASK null New in version 1.3!
The MASK variable gives the ultimate flexibility in populating the selected date back to your input field. Using a string that you construct you can dictate the exact format that will be passed to your form field. For example, passing the string MASK:DDDD, MMMM d, YYYY would yield Friday, May 26, 2005. The available options are as follows:
D 1-2 Digit Date
d Date with suffix
DD 2 Digit Date
DDD First three letters of day
DDDD Full day name
M 1-2 Digit Month
MM Two Digit Month
MMM First three letters of Month
MMMM Full month name
YY Two digit year
YYYY Four Digit Year


Note that the full and partial month and day names are based on the FullDayNames and FullMonthNames array--which is completely configurable! Not all date formats can be correctly read back into DateClick when setting the default date. This mostly includes dates that have month or day name literals in them, such as August instead of 08. Also note that this feature overrides the FORMAT, ZEROS, and DELIMITER options on the Pro and Developer versions.

e.g. MASK:DD/MM/YYYY;
ZEROS true If set to true, months and days that are less than 10 will be preceded by a zero, so a date of 5/8/1979 would look like 05/08/1979. Note that this setting is overridden by what is set in MASK!


e.g. ZEROS:false;
MOUSEOVER false If set to true, the calendar will pop up when the mouse moves over it.  If set to false, the DateClick icon needs to be clicked to activate.

e.g. MOUSEOVER:false;
SHOWDAYS true Set to true by default, you can set this to false to hide the header row that shows the days.  The days are usually displayed as S, M, T, etc. but can be set to show Sun, Mon, Tue, etc. by modifying the ary_days array property.

e.g. SHOWDAYS:false;
DELAY 2000

Amount of time in milliseconds before a calendar becomes hidden again after a pop-up. A value of 2000 keeps a calendar visible for two seconds after the mouse leaves the calendar area.

e.g. DELAY:5000; (5 second delay)

BEGINONMONDAY false

The default DateClick calendar displays with Sunday being the first day of the week.  By setting this to 'true' the day sequence will go M, T, W, T, F, S, S (with Saturday and Sunday at the end of the week).

e.g. BEGINONMONDAY:true;

ICONLEFT null

By default, DateClick is positioned on your web page relatively based on where in the code you call the write_calendar method.  You can also use absolute positioning in supported browsers by specifying the ICONLEFT and/or ICONTOP properties.  The size should be in pixels and denotes the amount of pixels from the left side of the screen the DateClick icon should appear.

e.g. ICONLEFT:40;

ICONTOP null

Same as above, but specifies the distance in pixels from the top of the page.

e.g. ICONTOP:20;

MOVEMODE 0

Tells DateClick whether the DHTML calendar can be moved by dragging it with the mouse. The default setting is 0 which means it can not be moved. Setting this to 1 will allow the user to move the calendar at their will. A setting of 2 is similar to 1, but if the calendar goes away and is invoked again its position will be reset to where the calendar icon is.

e.g. MOVEMODE:2;

MONTH 0

Controls how the month flyout is shown. The default 0 does not allow the month selection flyout to appear. If set to 1, the flyout will appear when the month in the title is clicked. If set to 2, the flyout will appear when the mouse moves over the month.

e.g. MONTH:2;

YEAR 0

Controls how the year flyout is shown. The default 0 does not allow the year selection flyout to appear. If set to 1, the flyout will appear when the year in the title is clicked. If set to 2, the flyout will appear when the mouse moves over the year.

e.g. YEAR:1;

HIDELISTBOXES false

One of the unfortunate side effects of the way browsers handle select and drop down lists is that they will always be rendered by the page on top of any DHTML elements. This means if they are in the same area as the DateClick calendar, they will show through. If this is likely to occur on your page, set HIDELISTBOXES to true. DateClick will detect any list boxes on the page and make them temporarily invisible.

e.g. HIDELISTBOXES:true;

SHADOW false

When set to true, applies a drop shadow to the calendar on compatible browsers (IE only at this time).

e.g. SHADOW:true;

CLOSE true

When set to false, the calendar will not close when a date is selected.

e.g. CLOSE:false;

INVALID 0

By default, dates in the past can be selected. Set INVALID to 1 and today and anything after today can be selected. Set to 2 and the first valid day begins tomorrow.

e.g. INVALID:1;

DIR basic Probably the most powerful setting in DateClick's configuration.  By specifying a DIR parameter, you set a specific class prefix and image directory upon which to base the appearance of DateClick's icon and calendar pop-up.  The main purpose of this is to allow you to customize DateClick's appearance and to allow multiple DateClick calendars on each screen all with a different look. By setting this to a setting other than basic, you can create a whole new CSS class structure prefixed by the chosen word that completely alters the appearance.  Also create a subdirectory with the name of your setting and place the calendar icon and supplemental images within it.
STATIC 0 Set this to 1 to set the popup calendar to "static" mode--that is DateClick always shows on the page without having to click a calendar icon to bring it up.

e.g. STATIC:1;

FUNCTION Pass the name of a function with this parameter that will be called when a date is selected. Don't include the ()'s in this declaration. You must then define the function with one argument--this argument will be a Date object set to the selected date. If a FIELD is passed as well as function, both the field will be set and the function will be called.

e.g. FUNCTION:showDate;

POSITION 0 Determines how the popup calendar will be positioned when it appears in non-static mode.
  • 0 - Default, calendar appears at the point of the mouse click.
  • 1 - Calendar appears at the location of the calendar icon.
  • 2 - Manual position. Starts at the top left and can be offset by X and Y parameters.

e.g. POSITION:2;

X and Y 0 Specifying X and Y in the configuration string will offset the popup calendar's screen position horizontally, with the X parameter, and vertically with the Y parameter. For example, if your POSITION parameter is set to 1, and the X is set to 10 and the Y is set to 15, the string would look like POSITION:1;X:10;Y:15; and the result would be that when the calendar pops up, its location will be 10 pixels to the right and 15 pixels down from the location of the icon image. The position can also be moved up or to the left by specifying negative values.

e.g. X:200;

TOOLTIP Pass text to the tooltip parameter to replace the default "Click here..." message that shows when the mouse hovers over the DateClick calendar icon.

e.g. TOOLTIP:Click here to display a popup calendar;

List of CSS Classes

DateClick relies on a number of CSS classes to determine its appearance as it is displayed on the page.  By setting the DIR parameter in the configuration string and preceding your class names with the value specified, you can include any number of DateClick calendars on your page with any different settings.  The classes that are used to determine the look of DateClick are listed below:

basic-cl-header The display settings for the month and year text in the header of the calendar.
basic-cl-on-month Style of the dates that are in the currently selected month.
basic-cl-on-month-lit Style of a date when the mouse is over it.
basic-cl-off-month Dates that are visible in the calendar but not in the currently selected month.
basic-cl-off-month-lit Same as above with mouse over.
basic-cl-body Style of the main calendar body.
basic-cl-year Style for each year in the year fly-out.
basic-cl-year-lit Same as above with mouse over.
basic-cl-day Format of each of the days in the day header row.
basic-cl-days Format for the day header row.
